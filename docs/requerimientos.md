# 2. Requerimientos Fundamentales

### En función del sistema informático seleccionado, menciona los cinco (05) requerimientos fundamentales del programa de instalación para garantizar que éste funcione correctamente.

Para garantizar que el sistema de información de Santiago Seguros, desarrollado en Python con MySQL, funcione correctamente una vez instalado, el programa de instalación (creado con Inno Setup) debe verificar y gestionar de manera rigurosa los siguientes cinco requerimientos fundamentales, los cuales constituyen las condiciones mínimas que debe cumplir el equipo destino para soportar la aplicación.

**1. Requerimiento de Sistema Operativo y Arquitectura.**  
El instalador debe verificar que el equipo destino ejecute una versión del sistema operativo Windows que sea compatible tanto con el intérprete de Python como con el servidor de base de datos MySQL. Dado que Inno Setup soporta todas las versiones modernas de Windows, el instalador deberá detectar específicamente si el sistema es de 32 o 64 bits. Esta detección es crítica, ya que determinará qué versión del conector MySQL (por ejemplo, `mysql-connector-python` o `PyMySQL`) y qué variante del runtime de Python (si se empaqueta una versión portátil) se deben instalar o configurar para asegurar la máxima compatibilidad y rendimiento.

**2. Requerimiento de Memoria RAM y Espacio en Disco.**  
Para que el sistema de información funcione de manera fluida, el instalador debe comprobar que el equipo cumple con los requisitos mínimos de hardware. Si bien el documento no especifica cifras para este caso particular, un análisis descriptivo de clientes y seguros puede consumir una cantidad considerable de recursos. El instalador debería verificar que existe al menos 4 GB de RAM disponible, siendo lo recomendado 8 GB para un funcionamiento más fluido, especialmente al manejar consultas complejas sobre la base de datos de clientes. En cuanto al almacenamiento, se debe verificar un espacio mínimo de 500 MB libres en el disco duro. Este espacio será necesario para albergar el código de la aplicación Python, el servidor MySQL (especialmente si se opta por una instalación local y no remota), los archivos de datos de la base de datos (que crecerán con el registro de clientes y sus pólizas), y los archivos de registro (logs) del sistema.

**3. Requerimiento de Software Base: MySQL Server.**  
El sistema de información depende absolutamente de MySQL para la persistencia de los datos (Rut, nombre, tipo de seguro, ingresos anuales, etc.). Por lo tanto, el programa de instalación tiene dos opciones, y su diseño debe definir cuál implementa. La primera y más robusta es verificar si MySQL Server ya está instalado en el equipo o en un servidor accesible en la red de Santiago Seguros. Si no lo está, el instalador debe tener la capacidad de ejecutar silenciosamente el instalador de MySQL Community Server (previamente empaquetado o descargado), configurando una instancia local con un usuario y contraseña predeterminados (por ejemplo, `santiago_user` / `seguros2024`), y creando automáticamente la base de datos y las tablas necesarias ejecutando un script SQL desde el instalador. La segunda opción, más simple pero menos común, es empaquetar una versión "portable" o "embebida" de MySQL dentro del propio instalador, asegurando que la aplicación sea completamente autónoma.

**4. Requerimiento de Software Base: Entorno de Ejecución Python.**  
De manera similar a MySQL, el sistema, al estar escrito en Python, requiere de un intérprete para ejecutar su código. El instalador debe verificar la presencia de Python 3.x en el sistema. Dado que es muy probable que un usuario de negocio en una aseguradora no tenga Python instalado, el programa de instalación (creado con Inno Setup) debería incluir la capacidad de instalar una versión específica y ligera de Python (por ejemplo, la 3.8 o superior) de manera desatendida, o bien, utilizar herramientas como PyInstaller para empaquetar toda la aplicación Python en un solo archivo .exe, eliminando así la dependencia de una instalación global de Python en el equipo del cliente.

**5. Requerimiento de Conectores y Bibliotecas (Drivers).**  
Más allá del intérprete de Python, la aplicación necesita las bibliotecas específicas para conectarse a MySQL. El instalador debe asegurar la presencia de los "controladores de dispositivos y complementos", como se menciona en el material de estudio. Para el caso de Python y MySQL, esto se traduce en bibliotecas como `mysql-connector-python` (la oficial de Oracle) o `PyMySQL`. El instalador debe ejecutar un comando como `pip install mysql-connector-python` dentro del contexto de la instalación de Python del sistema. Además, si la aplicación utiliza bibliotecas para análisis de datos (fundamental para un "análisis descriptivo" de ingresos y tipos de seguro), como `pandas` o `matplotlib`, el instalador también debe verificar su existencia e instalarlas automáticamente como parte del proceso de configuración.
